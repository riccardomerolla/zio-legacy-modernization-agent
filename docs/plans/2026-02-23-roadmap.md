# Product Roadmap

**Date:** 2026-02-23
**Status:** Active

## Completed Foundation

The persistence layer has been fully redesigned (#299-#306):
- EclipseStore with event sourcing for core aggregates (TaskRun, Conversation, Issues)
- Direct state storage for config (Settings, Workflows, CustomAgents)
- Opaque String IDs (UUID-based), no more Long↔String conversions
- BCE package structure (Boundary/Control/Entity) across all domains
- ADT-based state modeling (no Option bags for state-dependent fields)

## M1: Core Platform

**Goal:** Multi-channel, multi-agent operation with proper session and model management.

| # | Issue | Priority | Dependencies |
|---|-------|----------|--------------|
| #268 | Multi-channel management (Discord, Slack, WhatsApp) | High | — |
| #272 | Models management (multi-provider, fallbacks, per-agent overrides) | High | — |
| #274 | Session management (list, inspect, resume sessions) | Medium | — |
| #276 | Multi-agent routing (@agent addressing, channel bindings) | High | — |

**Suggested order:**
1. **#272 Models management** — foundation for all AI interactions, enables fallback chains
2. **#276 Multi-agent routing** — @agent syntax + channel bindings, core UX improvement
3. **#274 Session management** — session visibility, depends on conversation event store
4. **#268 Multi-channel** — Discord + Slack implementations, extends gateway architecture

**Key outcome:** Users can target specific agents via @syntax, manage AI models with fallbacks, monitor active sessions, and connect via Discord/Slack alongside Telegram.
